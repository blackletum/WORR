# FR-10-T10 Headless Moving-Brush Runtime Gate

Date: 2026-07-15

Project task: `FR-10-T10`

## Purpose

This gate closes the gap between the isolated immutable inline-BSP parity
fixture and a live server collision query.  It proves one deliberately narrow,
real moving-brush scenario: a sealed historical mover pose blocks a shot after
the authoritative live mover has changed position, while the query leaves the
authoritative collision world unchanged.

It is a dedicated-server-only test. It does not launch a graphical client,
renderer, or interactive UI, and it does not claim normal player-on-mover
physics, rotating, or complex-map fairness coverage.

## Packaged fixture

`assets/maps/worr_fr10_rewind_mover.bsp` is generated by the existing
`tools/networking/generate_rewind_collision_bsp_fixture.py` generator and is
included as a loose staged map by `tools/package_assets.py`.  Its SHA-256 is
`42642d702c39f5d7a4d788964c4a9e9e5e7d92c5cc3cf45b953cef987fe6fe3d`.

The fixture contains a real start-on `func_rotating` `SOLID_BSP` push brush,
a water inline model, and four separated `info_player_deathmatch` spawns,
unlike the existing renderer-oriented first-frame model fixture. The spawns permit the
dedicated runtime gate to bind its rider provenance check to a real engine bot
and validate ordinary rotating-pusher frames while also admitting two live
clients across an in-session reconnect without spawn occupancy ambiguity.
The gate discovers a live eligible mover; it does not branch on a map name or
manufacture an engine collision asset.

## Runtime probe

`LagCompensation_RunHistoricalBrushRuntimeProbe` is exposed only as a
diagnostic test seam and is invoked with `sv worr_rewind_mover_selftest`.
The probe:

1. Captures the actual live mover through the normal bounded pose-history path
   and verifies that its current map-epoch asset can be resolved.
2. Seals the corresponding canonical historical brush context and confirms a
   direct current-world collision against the original live mover.
3. Moves the live mover +96 units on X only after the historical context has
   been sealed.  The sealed context omits that mover from its current-world
   baseline, so the baseline trace must be unblocked.
4. Calls the immutable engine collision provider at the sealed historical
   transform.  The provider must dispatch and block the same ray.
5. Compares the authoritative collision fingerprint before and after the
   probe, then restores the actual entity origin and link state through RAII.

The status cvar `sg_worr_rewind_mover_selftest_status` has a stable,
machine-readable `pass`/`fail` schema.  It records setup and scene readiness,
reference collision, baseline exclusion, provider dispatch and historical
block, authority preservation, candidate count, quantized fractions, and a
failure code. Schema V5 additionally requires a rotation-applied flag, an
unrotated immutable-asset negative control, and a real bot rider whose normal
server-frame history and final sealed pose preserve mover-relative provenance,
and that the newest normal-frame pair enters an exact sealed canonical scene;
see
`docs-dev/fr-10-t10-rotated-moving-brush-runtime-gate-2026-07-15.md` and
`docs-dev/fr-10-t10-live-player-on-mover-provenance-gate-2026-07-15.md` and
`docs-dev/fr-10-t10-normal-frame-rider-continuity-gate-2026-07-15.md`.
The external gate rejects partial, duplicate, stale, or inconsistent status
rows.

## Rewind ABI correction

The probe exposed a real production bug in the optional mover reference:
`AbsentEntityRef()` had used `WORR_EVENT_NO_ENTITY` as its index.  That value is
valid in event payloads but not in a rewind `worr_entity_ref_t`, whose absent
form is all zero.  As a result, pose validation treated ordinary player and
mover entries as invalid.  The helper now returns `{}` and `EntityFromRef()`
also rejects index zero before looking into the edict array.  This preserves
the fail-closed identity rule without overloading event semantics.
